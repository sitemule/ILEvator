iv_encode_base64_buffer

Integer iv_encode_base64_buffer ( Pointer output, Pointer input, Integer inputLength)

Base64 encodes the memory from the input pointer and outputs it to the output pointer. The caller needs to make sure that enough memory is available. The output size is around 33% bigger than the input size.

Parameters

  1. Pointer [value] — Output
  2. Pointer [value] — Input
  3. Integer (20) [value] — Input length

Return Value

Integer — Base64 encoded string

iv_encode_formUrl

Varchar iv_encode_formUrl ( Varchar unencodedValue)

Form URL encodes the passed value. The input string can contain single and multibyte characters. The space sign will be encoded as a plus sign.

Parameters

  1. Varchar (1048576) [value] — Input string

Return Value

Varchar — Form URL encoded string

iv_encode_mime

Varchar iv_encode_mime ( Varchar data, Varchar charset, Char encoding)

The passed value will be mime encoded. This procedure only supports base64 at the moment.

Parameters

  1. Varchar (524284) [value] — Value to be encoded
  2. Varchar (20) [nopass, omit, const] — Character set of the passed value (default: IV_ENCODE_MIME_UTF8)
  3. Char (1) [nopass, value] — Encoding scheme (default: IV_ENCODE_MIME_BASE64)

Return Value

Varchar — MIME encoded value

iv_encode_query

Varchar iv_encode_query ( Varchar key, Varchar value)

URL encodes the passed key and value part of a query string and concats them with an equal sign so that it can be safely used as a URL query parameter.

Parameters

  1. Varchar (1000) [value] — Query parameter key
  2. Varchar (32766) [value] — Query parameter value

Return Value

Varchar — URL encoded query parameter: encoded_key=encoded_value
